# CLI Reference

The `ci` command-line tool provides three main workflows:

- **`ci runner`**: Execute pipelines locally in Docker, native, or other drivers
- **`ci set-pipeline`**: Store pipelines on a remote server (requires a running `ci server`)
- **`ci server`**: Start an HTTP server that manages pipelines and executes them on demand

Browse commands below, or use `ci <command> --help` for quick reference.
